# Documentation

### Step 1: Make Changes to Your Code

1. **Modify Your Flask App**:
    - Make any necessary updates or changes to your `app.py`, HTML templates, or other files in your project.
2. **Update Dependencies (if any)**:
    - If you have added new dependencies, make sure to update your `requirements.txt` file:

        ```
        pip freeze > requirements.txt
        ```


### Step 2: Build and Deploy the Updated App

1. **Build the New Docker Image**:
    - Open a terminal in your project directory and build a new Docker image:

        ```
        gcloud builds submit --tag gcr.io/padayon-ko-gemini/flask-app
        ```

2. **Deploy the New Image to Cloud Run**:
    - Deploy the newly built image to Cloud Run:

        ```
        gcloud run deploy flask-app --image gcr.io/padayon-ko-gemini/flask-app --platform managed --region asia-southeast1

        ```


### Step 3: Verify the Update

1. **Check the Deployed App**:
    - Open your browser and navigate to the URL provided by Cloud Run to ensure your updates are live.

### Optional: Roll Back to a Previous Version

If you need to roll back to a previous version, you can use Cloud Runâ€™s revision management.

1. **List Revisions**:
    - List all revisions of your service:

        ```
        gcloud run revisions list --service flask-app --region YOUR_REGION

        ```

2. **Rollback to a Previous Revision**:
    - Update the service to use a previous revision:

        ```
        gcloud run services update-traffic flask-app --to-revisions REVISION_NAME=100 --region YOUR_REGION

        ```
